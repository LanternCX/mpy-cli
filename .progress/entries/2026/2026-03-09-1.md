## Title
Cache scanned ports for list command

## Background / Issue
`mpy-cli list` on developer machines with many mixed serial ports still spent too much time probing irrelevant ports even after concurrent probing and timeouts. The desired behavior changed to prefer previously seen ports, skip stale cached ports that are not currently available, and remain compatible with Windows-style `COM*` names.

## Actions / Outcome
- Approach 1: Probe every current port concurrently with a short timeout -> improved some cases but still wasted time on irrelevant ports.
- Approach 2: Filter cached ports by `/dev/*` existence -> rejected because it would break Windows compatibility.
- Final approach: Persist previously seen ports in `runtime.db`, intersect cached ports with the current `mpremote connect list` result, probe cached-and-current ports first, then fall back to full current-port probing only when needed -> keeps default scans fast while remaining cross-platform.

## Lessons / Refinements
- For serial discovery, "currently available" should come from the active backend scan result, not from OS-specific path heuristics.
- A small runtime cache can remove most redundant probing without changing user-visible device listing semantics.
