## Date
2026-03-11

## Title
Override using-git-worktrees with project git workflow

## Background / Issue
仓库已有项目级 `git-workflow`，明确默认不使用 superpowers 自带 worktree 工作流。但上游 superpowers 仍内置 `using-git-worktrees`，其默认行为会创建隔离 worktree，与本项目 Git 约束冲突。需要在项目内提供同名覆盖入口，避免代理继续命中内置 worktree 流程。

## Actions / Outcome
- Approach 1: 仅强化 `.opencode/skills/git-workflow/SKILL.md` -> 无法拦截对 `using-git-worktrees` 的直接命中。
- Approach 2: 新建 typo/别名技能 -> 不能覆盖 superpowers 同名入口，不满足“完全覆盖”。
- Final approach: 新建项目内同名 `.opencode/skills/using-git-worktrees/SKILL.md`，并将其定义为覆盖入口 -> 成功把 `using-git-worktrees` 重定向到项目 `git-workflow`，同时补充 RED 证据和交叉引用。

## Lessons / Refinements
- 当仓库需要压过 superpowers 的默认行为时，项目内同名技能比额外别名更可靠。
- 覆盖型技能应尽量只做“拦截 + 重定向”，避免重复复制目标技能的完整规则。
