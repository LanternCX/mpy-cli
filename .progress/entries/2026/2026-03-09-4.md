## Title
Reset scan records and keep only last successful snapshot

## Background / Issue
The cache semantics for `mpy-cli list` still accumulated historical successful ports, while the desired behavior was to cache only the most recent scan's successful results. A user-facing reset capability was also needed so previous scan records could be cleared before re-running discovery.

## Actions / Outcome
- Approach 1: Reuse the existing success marker as an append-only history -> caused stale successful ports to remain in stage-one known scans.
- Approach 2: Add regression tests for reset order, single-scan success snapshots, and cache clearing -> confirmed both data-model and CLI UX gaps.
- Final approach: Add `clear_scan_records()` to remove prior scan history, make `mark_scanned_port_successes()` replace the previous success snapshot on every run, and add `mpy-cli list --reset-scan-records` to clear records before scanning -> matches the intended UX and keeps known-first focused on the latest successful device set.

## Lessons / Refinements
- "Last successful scan" semantics require replacement, not accumulation.
- Reset features should clear both the broad history and the derived cache to avoid partial-state surprises.
