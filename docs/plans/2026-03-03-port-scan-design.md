# 端口自动扫描与选择设计文档

## 背景

当前 `mpy-cli` 在未传 `--port` 时，会直接要求用户手动输入串口。该流程可用但效率低，尤其在设备端口可被自动发现时仍需重复输入。

目标是提升交互体验：在交互模式下自动扫描可用设备端口，并让用户选择。

## 目标

- 仅在交互模式下触发自动扫描。
- 扫描命令使用 `mpremote connect list`。
- 端口选择策略：
  - 0 个端口：提示并回退手动输入。
  - 1 个端口：默认端口并确认是否使用。
  - 多个端口：使用 `questionary.select` 供用户选择。
- 非交互模式保持原行为：未提供 `--port` 时直接报错。

## 非目标

- 不新增独立 `ports` 子命令。
- 不在非交互模式自动挑选第一个端口。
- 不改变部署计划与执行逻辑。

## 设计方案

### 模块改动

- `mpy_cli/backend/mpremote.py`
  - 新增 `list_ports()`，封装 `mpremote connect list`。
  - 新增输出解析函数，提取候选串口列表。

- `mpy_cli/cli.py`
  - 新增端口解析流程函数，优先级为：
    1. `--port`
    2. 配置文件 `serial_port`
    3. 交互扫描选择
    4. 手动输入
  - `--no-interactive` 保持严格：若前两项都无值，报错。

### 错误处理

- 扫描命令失败时不中断主流程，回退手动输入。
- 扫描结果为空时回退手动输入。

## 测试策略（TDD）

- `tests/test_mpremote_backend.py`
  - 验证端口解析函数对典型 `mpremote connect list` 输出的解析正确性。

- `tests/test_cli.py`
  - 验证交互模式下会优先扫描并选择端口。
  - 验证扫描为空时回退手动输入。
  - 验证非交互模式无端口时仍报错。

## 验收标准

- 用户在交互模式下不再必须先手动输入端口。
- 现有测试保持通过，新增测试覆盖扫描路径。
- 文档说明补充自动扫描行为（README Quick Start）。
